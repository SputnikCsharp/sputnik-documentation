## Overview

**Project:** NeoGenesis Park — Dinosaur Registration System  
**Team size:** 4 members  
**Stack:** .NET 9 · C# · Entity Framework Core · PostgreSQL · Rider IDE  
**Repository:** [neogenesis-park](https://github.com/Luismigonza/neogenesis-park)

| # | Role | Responsibilities |

---

## Branching Strategy

```
main       ← final delivery (production)
develop    ← integration branch (all PRs merge here)
feat/*     ← individual feature branches
```

**Rules:**
- Nobody commits directly to `main` or `develop`
- Each member works on their own `feat/` branch
- Pull Requests go from `feat/*` → `develop`
- The Architect (Person 1) reviews and approves all PRs
- At the end of the project, one final PR goes from `develop` → `main`

Before starting, make sure you have the following installed:

## Repository Setup (Person 1 — Architect)

### 1. Create the repository on GitHub

- **Name:** `neogenesis-park`
- **Visibility:** Public or Private (as required)
- **Initialize with:** README ✓ and `.gitignore` → select **VisualStudio**

### 2. Create branches

Create the following branches from GitHub after the repo is created:

```
main     ← created by default
develop  ← create manually from main
```

### 3. Clone the repository

```bash
git clone https://github.com/Luismigonza/neogenesis-park.git
cd neogenesis-park
```

### 4. Create the .NET 9 console project

```bash
dotnet new console -n NeoGenesisPark --framework net9.0
cd NeoGenesisPark
```

### 5. Install NuGet packages

Make sure you are inside the `NeoGenesisPark/` folder (where the `.csproj` file is located) before running these commands:

```bash
dotnet add package Microsoft.EntityFrameworkCore --version 9.0.1
dotnet add package Microsoft.EntityFrameworkCore.Design --version 9.0.1
dotnet add package Npgsql.EntityFrameworkCore.PostgreSQL --version 9.0.4
dotnet add package Microsoft.Extensions.Configuration --version 9.0.1
dotnet add package Microsoft.Extensions.Configuration.Json --version 9.0.1
```

> **Important:** Package versions must match the .NET version in use. Using .NET 9 requires version 9.x.x packages. Installing without specifying a version may pull incompatible v10 packages.

Verify the `.csproj` file looks like this:

```xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>net9.0</TargetFramework>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.EntityFrameworkCore" Version="9.0.1" />
    <PackageReference Include="Microsoft.EntityFrameworkCore.Design" Version="9.0.1">
      <IncludeAssets>runtime; build; native; contentfiles; analyzers; buildtransitive</IncludeAssets>
      <PrivateAssets>all</PrivateAssets>
    </PackageReference>
    <PackageReference Include="Microsoft.Extensions.Configuration" Version="9.0.1" />
    <PackageReference Include="Microsoft.Extensions.Configuration.Json" Version="9.0.1" />
    <PackageReference Include="Npgsql.EntityFrameworkCore.PostgreSQL" Version="9.0.4" />
  </ItemGroup>
</Project>
```

Confirm everything builds correctly:

```bash
dotnet build
# Expected: Build succeeded. 0 Warning(s). 0 Error(s)
```

### 6. Create the project folder structure

Inside `NeoGenesisPark/`, create the following folders manually in Rider (right-click → New Folder):

```
NeoGenesisPark/
├── Models/         ← Person 1
├── Data/           ← Person 1
├── Modules/        ← Persons 2 and 3
├── Validations/    ← Person 2
├── Seed/           ← Person 4
├── Helpers/        ← Person 4
├── Migrations/     ← auto-generated by EF Core (do not create manually)
├── appsettings.json
├── Program.cs
└── NeoGenesisPark.csproj
```

### 7. Configure `.gitignore`

The GitHub-generated `.gitignore` for VisualStudio already covers `bin/`, `obj/`, and `*.env`. Add the following line at the end of the file for Rider:

```gitignore
# Rider
.idea/
```

### 8. Configure `appsettings.json`

Create `appsettings.json` inside `NeoGenesisPark/` with placeholder values — **never commit real credentials**:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=IP_DEL_VPS;Database=neogenesis_park;Username=TU_USUARIO;Password=TU_CONTRASEÑA"
  }
}
```

Each team member creates a local `appsettings.Development.json` file with their real VPS credentials. This file is already covered by `.gitignore` and will never be pushed to the repository.

### 9. First commit and push

```bash
git checkout -b feat/modelo
git add .
git commit -m "chore: estructura inicial del proyecto y configuracion base"
git push origin feat/modelo
```

### 10. Open a Pull Request to `develop`

- Go to GitHub → your `feat/modelo` branch
- Click **Contribute → Open pull request**
- Make sure the target is `feat/modelo` → `develop` (not `main`)
- Title: `chore: estructura inicial del proyecto`
- Approve and merge the PR (Person 1 is the reviewer)

---

## How Other Team Members Get Started

Once Person 1 merges the initial PR to `develop`, every other team member does the following:

```bash
git clone https://github.com/Luismigonza/neogenesis-park.git
cd neogenesis-park
git checkout develop
git checkout -b feat/their-module-name
```

Then each person creates their own `appsettings.Development.json` locally with the real VPS credentials before running the project.

## Conventional Commits Reference

| Prefix | When to use |
|--------|-------------|
| `feat:` | Adding a new feature or module |
| `chore:` | Configuration, setup, packages |
| `fix:` | Bug fixes |
| `docs:` | Documentation changes |
| `refactor:` | Code restructuring without changing behavior |

**Examples:**
```
feat: agregar modulo de insercion de dinosaurios
chore: configurar ef core y migraciones
fix: corregir validacion de email duplicado
docs: agregar documentacion del modulo de consultas
```
